Git essentially is a change tracking software, it will create a folder inside the working folder
to store all changes made. Github is just a backup website for storing these changes.

The main use of it is just to track changes.

Remotes: Cloud-focus mindset. The main repo is the one on the cloud (Github), we pull the repo to 
our local computer and make changes and push it back.

Staging area: Not all changes need to be documented. Staging area allows one to control what 
changes are to be documented.

Branches: Different version of the main repo. Example, when one wants to add a new feature,
one could create a branch of the main repo, then after finishing the feature, then  
merge the branch to the main branch.

Do not create a git folder inside an already git folder. It's not that you can't, just don't do it.

Making changes directly on the Github page is generally not recommended. But for the purpose of the workshop
we do this for demonstration.

To track history (say one commits new code and now the codebase doesn't work) one can go back to the 
history of previous commit by the command

`git checkout HEAD-n` with n being the previous n commits.
